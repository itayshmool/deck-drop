The Problem

Why Context Documents Matter

The Wolf analyzes your org structure (headcount, reporting lines, departments). But structure alone doesn't tell the full story.

Without Context
  • "Department X has a 2.1:1 builder ratio"
  • "3 layers of management in team Y"
  • "15% contractor headcount"

Raw metrics without meaning.

With Context
  • "Dept X is being merged per the 2026 plan"
  • "Team Y has key-person risk on its lead"
  • "Contractors are converting in Q2"

Insights grounded in strategy.

The Stack

Three Layers of Intelligence

L1

Org Data — CSV/Excel upload. Employee records, reporting lines, departments, titles, locations. The structural foundation.

L2

Performance Data — KPIs, velocity metrics, attrition rates, engagement scores. Quantitative evidence layer.

L3

Context Documents — Strategic plans, QBR notes, culture docs, key-person risks, team context. The qualitative intelligence that makes analysis actionable.

Each layer makes the Wolf's analysis deeper. Context documents are what separate a generic org report from a tailored transformation plan.

Getting Data In

Two Ways to Add Context

File Upload

Drag and drop .pdf, .txt, .md, or .docx files. The system extracts text automatically.

PDFs are parsed with pdfjs-dist. Large documents are supported — the chunker handles any size.

Best for: Existing documents, reports, presentations exported as PDF.

Text Paste

Paste content directly into the text area. Give it a title and select a type.

Great for notes, summaries, or information scattered across Slack/email/docs.

Best for: Meeting notes, quick context, information you want to add without creating a file.

Both methods trigger the same RAG pipeline: chunk → embed → store → ready for retrieval.

At Analysis Time

How Context Gets Retrieved

Q

Query Built

Analysis scope converted to a natural language query. Department-scoped queries are narrower.

→
E

Query Embedded

Query text converted to a 1536-dim vector using the same model as documents.

→
S

Similarity Search

Cosine similarity via pgvector. Top 20 chunks retrieved, trimmed to token budget.

→
I

Injected

Retrieved chunks grouped by type and injected into agent prompts as structured context.

The retriever respects scope filters: department-focused analyses only see chunks tagged to that department (plus global context). Token budget prevents prompt overflow.

Getting the Most Out of It

Context Best Practices

Do

  • Upload your annual/strategic plan — the single highest-impact document
  • Add key-person risk notes — even bullet points help the Wolf flag single-points-of-failure
  • Include QBR metric definitions — teaches the Wolf what "good" looks like in your org
  • Tag department context — set entity_type to scope documents for focused analyses
  • Re-upload when plans change — hit Reindex in the Control Room to refresh embeddings

Don't

  • Upload raw data exports — the chunker works on prose, not CSVs
  • Paste entire books — focused, relevant content retrieves better than dump-everything
  • Worry about formatting — the system strips to plain text before chunking
  • Skip context for focused analyses — department scopes benefit the most from targeted context
